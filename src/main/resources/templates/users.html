Cookie 認證說明：
下方的用戶數據通過 fetch('/api/users') 從後端接口加載。
由於前端頁面和後端接口是同源的（同一個域名），瀏覽器會自動在請求頭中帶上 JSESSIONID Cookie，無需任何手動操作。
後端 Spring Security 收到請求後，用 Cookie 裏的 Session ID 查找服務端 Session，識別用戶身份，然後放行接口。
你可以打開瀏覽器開發者工具（F12）→ Network 面板，找到 /api/users 請求，在 Request Headers 裏看到 Cookie: JSESSIONID=xxx。

前端代碼（本頁實際執行的邏輯）

// 不需要設置任何 Authorization 頭
// 瀏覽器同源請求會自動攜帶 Cookie（包括 JSESSIONID）
fetch('/api/users')
  .then(res => {
    // 如果未登錄，Spring Security 返回 302 跳轉，fetch 跟隨後得到登錄頁 HTML
    // 這裏 res.ok 為 false，說明 Session 已失效
    if (!res.ok) window.location.href = '/login';
    return res.json();
  })
  .then(data => renderTable(data));
